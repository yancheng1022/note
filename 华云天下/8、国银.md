# 1、目标检测评价指标


## 1.1、交并比

衡量预测框与真实框的重叠程度

```text
IoU = 预测框 ∩ 真实框的面积 / 预测框 ∪ 真实框的面积
```

- **阈值设定**：通常设为0.5（COCO数据集用0.5:0.95的多阈值）
- 如果IoU ≥ 阈值 → 被认为是正确检测（TP候选）


## 1.2、混淆矩阵的四个基本量

对于每个类别：

- **TP（真阳性）**：正确检测（IoU ≥ 阈值且分类正确）
- **FP（假阳性）**：错误检测（IoU < 阈值，或分类错误，或重复检测）
- **FN（假阴性）**：漏检（真实物体未被检测到）
- **TN（真阴性）**：背景正确识别（目标检测中一般不直接计算）

## 1.3、精确率（precision）与召回率（recall）

- **精确率 P = TP / (TP + FP)** - “找得准不准”
- **召回率 R = TP / (TP + FN)** - “找得全不全”

## 1.4、P-R曲线（精确率-召回率曲线）

通过改变**置信度阈值**得到的一系列(P, R)点绘制的曲线：

- 阈值高 → 精确率高、召回率低（保守）
- 阈值低 → 精确率低、召回率高（激进）

## 1.5、mAP（平均精度）

1. **AP（平均精度）**
- 对单个类别，**P-R曲线下的面积**
- 计算方式（两种主流）：
	- **VOC 2007**：在召回率0-1之间等间隔取11个点，计算精确率平均值
	- **VOC 2012+ / COCO**：对所有不同的召回率点，计算精确率的插值平均值


2. **mAP（mean Average Precision）**
- **所有类别的AP的平均值**
- YOLO论文和报告中**最常引用的指标**

3. **mAP的不同变体**
```text
mAP@0.5   : IoU阈值固定为0.5时的mAP
mAP@0.5:0.95 : IoU阈值从0.5到0.95，步长0.05，取10个阈值的平均值（COCO标准）
```

## 1.6、损失函数

损失函数是深度学习模型训练的核心驱动力，它定量地告诉模型“你的预测有多糟糕”，并指引模型向正确的方向改进。
想象你在学投篮：每次投篮：是你的“预测”，篮筐位置：是“真实值”，距离篮筐的远近：就是“损失值”，你的目标：让这个距离（损失）越来越小
